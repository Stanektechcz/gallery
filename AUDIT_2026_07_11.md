## Opravené chyby

| Oblast | Problém | Oprava |
|---|---|---|
| Statistiky | `/stats` padaly na SQLite kvůli `YEAR()` a `MONTH()`. | Přenositelné výrazy pro SQLite a MySQL. |
| Timeline | Agregace rok/měsíc/den používaly MySQL-only SQL. | Přenositelné datumové agregace, vzpomínky používají Eloquent date filtry. |
| Itinerář | `/api/v1/itinerary` padalo kvůli MySQL `FIELD()`. | Prioritní řazení přes SQL `CASE`; opravený výpočet unikátních oblastí. |
| Jízdenky | Křehké endpointy dopravců či deep-linky mohly skončit HTTP 404. | Živé načtení zůstává, ale vždy existuje bezpečný provider fallback pro RegioJet, FlixBus, ČD, IDOS, Leo Express a Omio. |
| Rozpis cesty | Body byly prakticky nepřesunutelné na mobilu; bloky itineráře měly pouze dekorativní úchyt. | Přidány ovládací prvky nahoru/dolů, bezpečný rollback a úprava bloků. |
| Mapy a místa | Lokální písmo se dostávalo do názvů míst; země nebyly konzistentně česky. | Český/latinský výběr názvů, transliterace a české názvy zemí z kódu země. |
| Navigace | Neaktivní předchozí/další médium vedlo na `#`; dopravní karta mohla vytvořit prázdný odkaz. | Neaktivní tlačítka bez odkazu, vykreslení jen validních booking URL. |
| Pozvánky | Administrátorská pozvánka nevytvořila e-mail a autor pozvánky se ukládal do chybně typovaného boolean sloupce. | E-mailová notifikace pozvánky, nový FK `invited_by_user_id`, bezpečný fallback odkaz jen při selhání mailu a pokrytí testem. |
| Plánování | Kalendář neuměl oddělit akce od cest, kontrolovat překryvy ani udržet přípravy pohromadě. | Společné akce, účastníci, checklisty, rezervace, opakování, rozpočty, varianty tras, inbox a upozornění. |
| Integrace | Po deaktivaci již nastaveného poskytovatele nebylo možné jej znovu zapnout bez opětovného zadání tajného klíče. | Zachování šifrované konfigurace při deaktivaci, bezpečné opětovné zapnutí a test pokrývající oba přechody. |

## Aktuální výsledek

V testovacím prostředí nebyla nalezena nefunkční statická podstránka, nevalidní interní odkaz ani čtecí API endpoint. Plná sada obsahuje **69 testů a 362 assertions**; produkční build PWA prošel.

## Provozní poznámky

- Pro plně české popisky map je připraveno `VITE_MAPY_API_KEY`. Bez klíče se používá latinkový mapový podklad, aby se neobjevovaly čínské či jiné nelatinkové popisky.
- Živé ceny dopravců jsou závislé na jejich veřejných rozhraních. Aplikace při jejich výpadku nepadá a nabídne funkční portály pro pokračování.
- E-mailové pozvánky a e-mailové připomínky vyžadují v produkci funkční `MAIL_MAILER`; při výpadku pozvánka nezanikne a administrátor dostane jednorázový zabezpečený odkaz.
- Serverový web-push vyžaduje VAPID klíče a HTTPS. Bez nich zůstávají spolehlivě dostupné in-app, PWA a e-mailové připomínky; aplikace netvrdí, že vzdálený push doručila.
- Vizuální E2E ověření v lokálním in-app prohlížeči nebylo možné provést, protože tento cíl je v lokální politice prohlížeče zakázán. Serverové route/API smoke testy a produkční build proběhly.
